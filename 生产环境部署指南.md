# JobCollection 生产环境部署指南（腾讯云轻量应用服务器 Ubuntu 22.04）

本文面向腾讯云轻量应用服务器（Ubuntu 22.04 LTS），采用非 Docker 方式部署：

- 前端：Vue + Vite 静态站点（Nginx 托管）
- 后端：FastAPI（Uvicorn）+ Celery Worker + Celery Beat
- 基础组件：PostgreSQL + Redis + Elasticsearch（推荐腾讯云托管 ES）
- 进程管理：`supervisor`

---

## 1. 部署前确认（腾讯云必做）

### 1.1 服务器配置建议

- 最低：2C4G（不建议本机部署 ES）
- 推荐：4C8G（可本机部署 ES 单节点）
- 系统：Ubuntu 22.04 LTS

### 1.2 域名与解析

- 准备业务域名，例如 `job.example.com`
- 在 DNS 里将 A 记录解析到轻量服务器公网 IP

### 1.3 腾讯云防火墙（轻量控制台）放通端口

- 必开：`22`（SSH）、`80`（HTTP）、`443`（HTTPS）
- 可选：`3000`（Grafana，仅内网或白名单开放）
- 不对公网开放：`5432`、`6379`、`9200`、`8000`、`9090`

## 3. 使用 SFTP 上传代码与目录约定

### 3.1 服务器准备目录

```bash
sudo mkdir -p /opt/job /opt/job/upload /opt/job/releases
sudo chown -R deploy:deploy /opt/job
```

### 3.2 本地打包（Windows PowerShell 示例）

在本地项目根目录执行（建议仅打包必要目录）：

```powershell
Set-Location d:\Code\job
Compress-Archive -Path .\frontend,.\jobCollectionWebApi,.\common,.\static,.\requirements.txt,.\prometheus.yml,.\grafana,.\deploy `
  -DestinationPath .\job_release.zip -Force
```

### 3.3 使用 SFTP 上传

```bash
sftp deploy@你的服务器公网IP
sftp> put d:/Code/job/job_release.zip /opt/job/upload/job_release.zip
sftp> exit
```

如果你用 WinSCP/FileZilla，也可以直接把 `job_release.zip` 上传到 `/opt/job/upload/`。

### 3.4 服务器解压发布

```bash
cd /opt/job
release_dir=/opt/job/releases/$(date +%Y%m%d_%H%M%S)
mkdir -p "$release_dir"
unzip -q /opt/job/upload/job_release.zip -d "$release_dir"
rsync -a "$release_dir"/ /opt/job/
```

目录约定（建议）：

- 项目根目录：`/opt/job`
- 上传目录：`/opt/job/upload`
- 发布历史：`/opt/job/releases`
- 后端目录：`/opt/job/jobCollectionWebApi`
- 前端目录：`/opt/job/frontend`
- 静态上传目录：`/opt/job/static`
- 前端发布目录：`/var/www/job/dist`

## 5. 环境变量配置（关键）

项目会优先读取 `.env.production`（当 `ENVIRONMENT=production`）或 `.env`。建议在 `/opt/job` 维护 `.env.production`。

```bash
cd /opt/job
cp .env.example .env.production
```

至少确认这些变量：

```ini
ENVIRONMENT=production
DEBUG=false
LOG_LEVEL=INFO

SECRET_KEY=请替换为强随机字符串
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=11520

BACKEND_CORS_ORIGINS=https://你的域名

POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=5432
POSTGRES_USER=job_user
POSTGRES_PASSWORD=请替换
POSTGRES_DB=job

REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=请替换

ES_HOST=你的 ES 地址（本机填 127.0.0.1）
ES_PORT=9200
ES_USER=elastic
ES_PASSWORD=请替换
ES_SCHEME=https

AI_PROVIDER=deepseek
AI_BASE_URL=https://api.deepseek.com
AI_API_KEY=请替换
AI_MODEL=deepseek-chat
```

注意：

- 不要把真实密钥提交到 Git 仓库。
- 如果历史文件里出现过真实密钥，建议立即在对应平台上轮换（Rotate）并失效旧密钥。

## 8. Elasticsearch 选择

### 方案 A（推荐）：腾讯云托管 Elasticsearch

- 在 `.env.production` 中填托管 ES 的 `ES_HOST/ES_PORT/ES_USER/ES_PASSWORD/ES_SCHEME`
- 轻量服务器无需本机安装 ES，资源压力更小

### 方案 B（本机部署，建议仅 4G+ 内存）

```bash
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo gpg --dearmor -o /usr/share/keyrings/elasticsearch-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/elasticsearch-keyring.gpg] https://artifacts.elastic.co/packages/8.x/apt stable main" \
  | sudo tee /etc/apt/sources.list.d/elastic-8.x.list
sudo apt update
sudo apt install -y elasticsearch
```

建议设置 JVM 堆（示例 1G）：

```bash
echo "-Xms1g" | sudo tee /etc/elasticsearch/jvm.options.d/heap.options
echo "-Xmx1g" | sudo tee -a /etc/elasticsearch/jvm.options.d/heap.options
sudo systemctl enable elasticsearch
sudo systemctl restart elasticsearch
```

## 12. Nginx 反向代理（完整配置文件，含 WebSocket）

如果你还没安装 Nginx，先执行第 `2.4` 节。

创建完整配置文件 `/etc/nginx/sites-available/job.conf`：

```nginx
# ==========================================================
# JobCollection Nginx Production Config
# 路径: /etc/nginx/sites-available/job.conf
# ==========================================================

# 1) HTTP: 统一跳转到 HTTPS
server {
    listen 80;
    listen [::]:80;
    server_name 你的域名;  # 例如 job.example.com

    location ^~ /.well-known/acme-challenge/ {
        root /var/www/html;
        allow all;
    }

    return 301 https://$host$request_uri;
}

# 2) HTTPS: 前端 + API + WebSocket + 静态文件
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name 你的域名;  # 例如 job.example.com

    # 证书路径（certbot 申请后替换为真实路径）
    ssl_certificate /etc/letsencrypt/live/你的域名/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/你的域名/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    root /var/www/job/dist;
    index index.html;

    charset utf-8;
    client_max_body_size 20m;
    server_tokens off;

    # 安全响应头（可按需调整）
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # 前端 SPA
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API + WebSocket
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    # 后端上传静态文件
    location /static/ {
        alias /opt/job/static/;
        expires 7d;
        add_header Cache-Control "public";
    }

    # 前端构建产物缓存
    location ~* \.(?:css|js|mjs|map|jpg|jpeg|gif|png|svg|ico|webp|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
}
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/job.conf /etc/nginx/sites-enabled/job.conf
sudo nginx -t
sudo systemctl reload nginx
```

如果你还没有证书，先用下面这个仅 HTTP 版本顶上（把上面的 `443` server 临时注释掉），等第 `13` 节申请证书后再切回完整配置：

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name 你的域名或公网IP;

    root /var/www/job/dist;
    index index.html;
    charset utf-8;
    client_max_body_size 20m;
    server_tokens off;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    location /static/ {
        alias /opt/job/static/;
    }
}
```

## 14. 部署 Prometheus + Grafana（监控）

### 14.1 安装 Prometheus

```bash
sudo apt install -y prometheus
sudo systemctl enable prometheus
```

编辑 `/etc/prometheus/prometheus.yml`，补充抓取你的后端指标：

```yaml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: "prometheus"
    static_configs:
      - targets: ["127.0.0.1:9090"]

  - job_name: "job-api"
    metrics_path: /metrics
    static_configs:
      - targets: ["127.0.0.1:8000"]
```

重启并检查：

```bash
sudo systemctl restart prometheus
sudo systemctl status prometheus
curl http://127.0.0.1:9090/-/ready
```

### 14.2 安装 Grafana

```bash
sudo apt install -y apt-transport-https software-properties-common
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://packages.grafana.com/gpg.key | sudo gpg --dearmor -o /etc/apt/keyrings/grafana.gpg
echo "deb [signed-by=/etc/apt/keyrings/grafana.gpg] https://packages.grafana.com/oss/deb stable main" | sudo tee /etc/apt/sources.list.d/grafana.list

sudo apt update
sudo apt install -y grafana
sudo systemctl enable grafana-server
sudo systemctl start grafana-server
sudo systemctl status grafana-server
```

默认地址与账号：

- 地址：`http://服务器IP:3000`
- 初始账号密码：`admin / admin`（首次登录强制改密）

### 14.3 Grafana 连接 Prometheus

在 Grafana 中添加数据源：

- Type: `Prometheus`
- URL: `http://127.0.0.1:9090`

建议导入你仓库下已有看板：

- `grafana/dashboards/job_platform_overview.json`
