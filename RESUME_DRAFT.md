# 项目经历：智能化高校专业就业分析平台

## 项目描述
这是一个基于海量招聘数据的高校专业就业前景分析平台。系统通过爬虫采集互联网招聘信息和高校专业数据，利用 Elasticsearch 和 MySQL 进行清洗与聚合分析，为大学生提供基于“专业-技能-薪资”维度的就业画像，帮助用户精准匹配职业发展路径。

## 技术栈
*   **后端**: Python 3.10, FastAPI (Async), SQLAlchemy (AsyncIO)
*   **数据存储**: MySQL 8.0, Redis (缓存/队列), Elasticsearch 8.x (搜索引擎)
*   **数据采集**: Scrapy, Aiohttp
*   **前端**: Vue 3, ECharts, Axios
*   **部署**: Uvicorn (Multi-workers), Docker

## 核心职责与产出 (Key Achievements)

### 1. 高并发后端架构设计
*   基于 **FastAPI** + **Async SQLAlchemy** 构建全异步后端服务，显著提升 I/O 密集型场景下的系统吞吐量。
*   设计并实施**多级缓存策略**：利用 **Redis** 缓存热点分析数据（如专业预设列表），减少 **90%** 的重复数据库查询；针对高并发场景调优 MySQL/Redis 连接池参数，配合多 Worker 模式部署，有效支撑高并发访问。

### 2. 搜索引擎与降级容灾
*   主导 **Elasticsearch** 聚合分析模块的开发，实现对百万级职位数据的毫秒级多维度聚合（薪资分布、核心技能、行业占比）。
*   设计**高可用降级方案**：在 ES 服务不可用时，利用 Python 装饰器模式无缝切换至 MySQL 进行 SQL 级兜底分析，确保核心业务 **100% 可用性**。

### 3. 高性能数据采集系统 (ETL)
*   重构 Scrapy 爬虫管道 (Pipeline)，设计 **异步批量写入缓冲区 (WriteBuffer)**，将单条插入优化为批量事务提交，解决了数据库写入瓶颈。
*   优化爬虫并发策略 (Semaphore/Concurrent Requests)，将单域名采集速度提升 **8倍**。
*   编写自动化 **ETL 脚本**，通过正则与关键词提取算法，将非结构化的“专业介绍文本”清洗为结构化的“专业-职位映射图谱”，实现了数据闭环。

### 4. 数据可视化前端
*   使用 **Vue 3** + **ECharts** 开发动态分析大屏，实现了从后端 API 获取复杂聚合数据并渲染为动态柱状图、饼图的完整链路，优化了用户交互体验。
